# CryptoBot v11.0 — Исправления и улучшения

## 🔧 Что исправлено (основная проблема: бот не торговал)

### 1. Слишком строгие фильтры сканера
- **Было**: ADX >= 15, ATR >= 0.5%, signal_strength >= 0.35, 2/3 MTF согласия
- **Стало**: ADX >= 12, ATR >= 0.3%, signal_strength >= 0.25, 1/3 MTF для микро-баланса
- **Результат**: бот теперь находит гораздо больше сигналов

### 2. Слишком мало символов сканировалось
- **Было**: 15 символов за скан, рандомная выборка
- **Стало**: 20-30 символов, приоритет по объёму торгов + рандом
- **Результат**: больше шансов найти подходящую пару

### 3. Индикаторы были слишком консервативными
- **Было**: нужно 4+ сигнала из 8, RSI 50-75 (бык) / 25-50 (медведь)
- **Стало**: нужно 3+ сигнала из 9, RSI 45-78 (бык) / 22-55 (медведь)
- **Добавлено**: ADX+DI/ADX-DI направление, EMA golden/death cross forming (0.5 балла)
- **Результат**: больше точек входа при сохранении качества

### 4. Риск-менеджер блокировал микро-балансы
- **Было**: risk_per_trade 0.5% для micro, max_total_risk 2%
- **Стало**: risk_per_trade 5% для micro, max_total_risk 10%, margin до 90% баланса
- **Результат**: с $19 можно открыть позицию на ~$17 (с плечом)

### 5. MTF требовал согласие даже когда данных нет
- **Было**: если MTF не ответил — сигнал отклонялся
- **Стало**: если MTF данных нет — требование снимается (effective_mtf_required = 0)
- **Результат**: меньше ложных отказов

### 6. Фильтры не адаптировались достаточно быстро
- **Было**: ослабление 0.85x за пустой скан
- **Стало**: ослабление 0.7x за пустой скан (агрессивный режим)
- **Результат**: после 1-2 пустых сканов фильтры сильно ослабляются

## 🧠 Улучшения "мозгов"

### Strategy Engine v11
- Отслеживание win/loss streak по стратегиям
- Снижение веса стратегий на проигрышной серии (>=3)

### Risk Manager v11
- Лучшая адаптация к микро-балансам (<$50)
- Anti-chase: увеличен лимит до 8 ордеров/час, cooldown 180 сек
- Более гибкое управление риском

### Exit Manager v11
- Исправлена логика частичного закрытия
- Корректное обновление SL после breakeven
- Более понятные логи

## 📊 Улучшения визуала и логов
- Все решения бота подробно объясняются в логах
- Формат: `DECISION: ACTION | param1=value1 | param2=value2`
- Статистика фильтров после каждого скана
- Приоритет сканирования: топ по объёму → случайные

## 🚀 Как обновить

1. Закрой бота (если запущен)
2. Скопируй все файлы из архива поверх существующих
3. Удали `__pycache__` папки:
   ```
   for /d /r . %d in (__pycache__) do @if exist "%d" rd /s /q "%d"
   ```
4. Запусти: `python main.py`
5. Убедись что в Config tab:
   - Demo Mode **СНЯТА**
   - API ключи введены
   - Scan interval = 3 минуты

## ⚠️ Важно
- Бот теперь агрессивнее ищет сигналы — это нормально для микро-баланса
- С $19 используй плечо 5-10x для достижения min notional
- Следи за логами — там объясняется каждое решение
